API Documentation

This page provides a concise, practical guide to integrating VerifyNow’s email verification API. You’ll find example requests in multiple languages, including cURL and Node.js Fetch, along with sample responses to illustrate the structure and expected data. All endpoints use OTP-based verification for secure user validation and can be orchestrated in your onboarding and authentication flows. For Hyderabad-based deployments, our default routing and security configurations help maintain low latency while preserving data integrity.

Endpoint: POST /v1/verify

Description: Verifies the provided email and returns a verification status along with OTP-ready metadata. Use this endpoint to initiate OTP-based verification as part of your onboarding or sign-up flow.

cURL

curl -X POST https://api.verifynow.example/v1/verify \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {{YOUR_API_KEY}}" \
  -d '{"email": "user@example.com", "otp": true}'

Node.js (Fetch)

const fetch = require('node-fetch');

fetch('https://api.verifynow.example/v1/verify', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_API_KEY'
  },
  body: JSON.stringify({ email: 'user@example.com', otp: true })
})
  .then(res => res.json())
  .then(json => console.log(json));

Sample Response

{
  "email": "user@example.com",
  "status": "verified",
  "otp_sent": true,
  "verification_id": "vx-1234abcd"
}

Notes: The response includes status, an OTP flag, and a verification_id for correlating with subsequent OTP validation requests. All data is transmitted securely over TLS 1.2+.
